# Fixture: xy-equals-15

A Spartan2 proof of the R1CS satisfiability statement
`x · y = target` with `target` as a public input (set to 15) and a
satisfying witness `x = 3, y = 5`. The R1CS instance is padded with
a trivial second constraint `z[0] · z[0] = z[0]` so the row count is
a power of two (m = 2). The variable layout is `z = (1, target, x, y)`,
total length n = 4, witness length 2.

The proof bytes are produced by the C# Spartan prover with a
deterministic scalar-random delegate seeded by the UTF-8 bytes of
`"veridical.spartan.fixture.xy-equals-15.v1"`. The deterministic RNG
is `Lumoin.Veridical.Tests.Spartan.DeterministicScalarRandom`
keyed by that seed.

Files:

- `instance.bin` — serialized R1CS instance. Layout is internal to the
  test project's `SpartanFixtureTests.SerialiseInstance`.
- `witness.bin` — canonical big-endian bytes of the witness scalars,
  in storage order (2 × 32 = 64 bytes).
- `proof.bin` — the Spartan2 proof bytes in the wire format described
  in `src/Lumoin.Veridical.Core/Spartan/SPARTAN2.md` §5.
- `transcript-domain.txt` — the Fiat-Shamir domain label used at
  prove and verify time.

The fixture is regenerated by running `RegenerateFixture` with the
environment variable `VERIDICAL_REGENERATE_FIXTURES=1`. The two
non-gated tests `XyEquals15FixtureProofVerifies` and
`CSharpProverProducesByteIdenticalFixtureProof` consume these bytes
and protect against silent wire-format drift across refactors.
