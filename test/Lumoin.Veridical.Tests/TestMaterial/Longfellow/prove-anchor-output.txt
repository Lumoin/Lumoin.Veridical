Wire-format-conformant Ligero PROVE ground truth from google/longfellow-zk (C.4 oracle).

Produced by prove_anchor.cc, run inside the longfellow-ref Docker image. The dump is computed by the
reference implementation in its own build environment via development tooling outside this repository:

  docker run --rm \
    -v "<repo>/tempdocs/longfellow-zk-reference:/src" \
    -v "<repo>/test/Lumoin.Veridical.Tests/TestMaterial/Longfellow:/anchor" \
    longfellow-ref bash -c \
    "g++ -O2 -std=c++17 -mpclmul -I /src/lib /anchor/prove_anchor.cc -lcrypto -o /tmp/a && /tmp/a"

The harness runs the REAL LigeroProver::commit()+prove() and the REAL LigeroProof, then cross-checks
with LigeroVerifier::verify(); _verify=1 why=ok confirms the dumped fields are a verifying proof.

Determinism: the harness injects a CounterRandomEngine whose k-th byte ever produced is (k & 0xFF),
identical to the commit anchor. The transcript is the Fiat-Shamir PRF, seeded with the fixed string
"c4". The linear constraints are nl = nw constraints, one term each: constraint c selects witness c
with coefficient of_scalar(c+1), and b[c] = coefficient * W[c] so the system is satisfiable. The
theorem statement hash_of_llterm is the fixed 32-byte pattern 0x10, 0x11, ..., 0x2f. The witnesses
are W[i] = of_scalar(i+1) with W[2] = W[0]*W[1] for the one quadratic constraint, nw=8 nq=1
rateinv=4 nreq=2.

The prove flow, in the order the transcript interleaves absorbs (P->V) and squeezes (V->P):
  1. commit: absorb the commitment root (write_commitment).
  2. absorb hash_of_llterm (the theorem statement).
  3. squeeze u_ldt (nwqrow elts), then the low-degree response y_ldt (block elts).
  4. squeeze alphal (nl elts), alphaq (3*nq elts), then the dot response y_dot (dblock elts).
  5. squeeze u_quad (nqtriples elts), then the quadratic response y_quad_0 (r elts) and y_quad_2
     (dblock-block elts). The middle w elts of y_quad are zero and not transmitted.
  6. absorb the four response rows (y_ldt, y_dot, y_quad_0, y_quad_2 as field-element arrays).
  7. squeeze idx (nreq distinct columns in [0, block_ext)), gather the opened columns req[nrow, nreq],
     open the Merkle proof (one nonce per opened column + the compressed multi-proof path).

Fields per prefix (q16 = production GF(2^16), q32 = test-parity GF(2^32)):
  *_param      : the derived LigeroParam dimensions.
  *_hashllterm : the 32-byte theorem statement absorbed before any challenge.
  *_b          : the linear-constraint targets b[c] = coefficient * W[c], the public inputs the
                 verifier's dot value-check compares (want_dot = sum_c b[c]*alphal[c]) against the
                 prover's response (proof_dot = sum_k y_dot[r+k]).
  *_root       : the commitment Merkle root (identical to the C.2 commit anchor's c16/c32 root).
  *_uldt/_alphal/_alphaq/_uquad : the challenge arrays exactly as squeezed, comma-separated, each
                 element 16 little-endian to_bytes_field bytes.
  *_yldt/_ydot/_yquad0/_yquad2  : the response rows, same element framing.
  *_idx        : the opened columns (decimal, in [0, block_ext)).
  *_reqI       : row I's nreq opened elements (the gathered column values at idx).
  *_nonceI     : the per-leaf nonce of opened column I (32 bytes).
  *_pathlen    : the number of digests in the compressed Merkle multi-proof.
  *_pathI      : the I-th multi-proof digest, in generate_compressed_proof selection order
                 (i = n-1 down to 1: for each tree node, the child NOT on a path to an opened leaf).
  *_verify     : 1 if LigeroVerifier::verify() accepts the proof.
